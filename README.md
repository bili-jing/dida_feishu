# 滴答清单 → 飞书多维表格 同步工具

将滴答清单（Dida365/TickTick）的任务数据同步到飞书多维表格，支持多用户、增量同步、附件处理。

## 功能特性

- **多用户管理** — 支持多账号，每个用户独立的飞书应用凭据
- **两种登录方式** — 手机号/邮箱密码登录、微信扫码登录
- **智能同步** — 首次全量同步，后续增量更新（只推送变化的任务）
- **附件处理** — 自动下载滴答附件并上传到飞书，带缓存避免重复处理
- **内容分类** — 自动识别任务类型（纯文本、聊天记录、图片、文件、链接、清单等）
- **多视图** — 自动创建「全部任务」「进行中」「已完成」「看板」视图
- **跨平台** — 支持编译为 Windows / macOS / Linux 独立可执行文件

## 环境要求

- [Bun](https://bun.sh/) v1.0+（开发运行）
- 飞书开放平台应用（用于同步到飞书）

## 安装

```bash
# 克隆项目
git clone <repo-url>
cd dida_feishu

# 安装依赖
bun install
```

## 配置

### 飞书应用凭据

可以通过以下任一方式配置：

#### 方式一：CLI 中绑定（推荐）

启动程序后，在「飞书配置」菜单中绑定 App ID 和 App Secret，凭据保存在本地数据库中。

#### 方式二：.env 文件

在项目根目录创建 `.env` 文件：

```env
FEISHU_APP_ID=cli_xxxx
FEISHU_APP_SECRET=xxxx
```

### 飞书应用创建步骤

1. 访问 [飞书开放平台](https://open.feishu.cn/app) 创建企业自建应用
2. 获取 App ID 和 App Secret
3. 在「权限管理」中开通以下权限：
   - `bitable:app` — 多维表格
   - `drive:drive` — 云空间（用于创建表格和上传附件）
4. 发布应用版本

## 使用方式

### 交互模式（推荐）

```bash
bun run dev
```

启动后进入交互式菜单：

```text
◆  请选择用户
│  ● 张三 (微信, 28天有效)
│  ○ 添加新用户
│  ○ 删除用户
```

#### 典型操作流程

```text
选择/添加用户 → 拉取最新数据 → 同步到飞书
```

#### 用户菜单

| 选项 | 说明 |
|------|------|
| **拉取最新数据** | 从滴答清单 API 拉取全部任务（活跃、已完成、已放弃、垃圾桶） |
| **数据概况** | 查看本地数据统计和飞书同步状态 |
| **搜索任务** | 按标题/内容关键词搜索，查看任务详情和同步状态 |
| **待同步任务** | 查看尚未同步或已修改的任务列表 |
| **同步到飞书** | 选择全量或增量同步，可选跳过附件 |
| **飞书配置** | 绑定/换绑/解绑飞书应用凭据 |

### 批量模式

无交互，直接拉取数据：

```bash
# 拉取所有已缓存用户的数据
bun run dev -- --user all

# 拉取指定用户
bun run dev -- --user user1@example.com,user2@example.com
```

### 同步脚本

独立的同步入口，适合定时任务或自动化场景：

```bash
# 自动选择模式（有配置→增量，无配置→全量）
bun run sync

# 强制全量同步（创建新表格）
bun sync.ts --full

# 跳过附件处理
bun sync.ts --no-attachments

# 同步指定用户
bun sync.ts --user user1@example.com

# 清除所有同步配置和缓存
bun sync.ts --reset

# 组合使用
bun sync.ts --full --no-attachments --user user1@example.com
```

| 截止时间 | 日期 | 截止日期 |
| 完成时间 | 日期 | 完成时间 |
| 滴答ID | 文本 | 原始任务 ID |

## 同步机制

### 全量同步

创建新的飞书多维表格，写入全部任务。适用于首次同步或需要重建表格。

### 增量同步

基于 `modified_time` 检测变化：

| 情况 | 操作 |
|------|------|
| 本地有、飞书没有 | 创建新记录 |
| `modified_time` 变化 | 更新飞书记录 |
| `modified_time` 未变 | 跳过 |
| 飞书表格不可访问 | 自动回退到全量同步 |

### 关于数据覆盖

同步方向是**单向**的（滴答清单 → 飞书）：

- 只在飞书改了（滴答没改）→ 安全，增量同步会跳过
- 飞书和滴答都改了同一个任务 → **飞书的改动会被覆盖**
- 建议以滴答清单为主数据源，飞书作为查看和分析平台

### 附件处理

1. 从滴答 API 下载附件到本地 `downloads/` 目录
2. 上传到飞书获取 `file_token`
3. 两步均有缓存，重复同步不会重复下载/上传

> 附件较多时处理可能耗时，可使用 `--no-attachments` 跳过。

## 配合飞书 AI 使用

同步完成后，可在飞书多维表格中添加「AI 字段捷径」列来增强数据：

| 需求 | 使用的捷径 | 引用字段 |
|------|-----------|---------|
| 总结聊天记录/长文本 | 内置「总结」 | 内容 |
| 分析 PDF/图片附件 | Kimi 阅读助手 | 附件 |
| 抓取链接网页内容 | AI 读取网页链接 (Webpilot) | 链接 |
| 自动分类/打标签 | 内置「分类」/「智能打标」 | 内容 |

> AI 字段捷径需在飞书 UI 中手动添加，开启「自动更新」后，新同步的数据会自动处理。

## 编译为可执行文件

```bash
# 编译所有平台
bun run build

# 单独编译
bun run build:win     # → dist/dida-feishu.exe
bun run build:mac     # → dist/dida-feishu-macos
bun run build:linux   # → dist/dida-feishu-linux
```

编译后的可执行文件：

- 不需要安装 Bun 或 Node.js
- 数据库和下载文件保存在可执行文件所在目录
- Windows 双击运行，退出前会等待按回车

## 数据存储

所有数据保存在本地 SQLite 数据库 `db/dida.db`：

| 表 | 说明 |
|----|------|
| `tokens` | 登录凭据缓存（30天有效期） |
| `projects` | 滴答清单项目/清单 |
| `tasks` | 任务数据 |
| `feishu_config` | 飞书表格配置 |
| `feishu_credentials` | 飞书应用凭据（per-user） |
| `sync_state` | 同步状态追踪 |
| `attachment_cache` | 附件下载/上传缓存 |

## 常见问题

**登录 token 过期了怎么办？**

Token 有效期约 30 天。过期后重新选择用户并「拉取最新数据」即可自动重新登录。

**Windows 下微信二维码乱码？**

已处理。Windows 环境会自动将二维码保存为 PNG 图片并用默认图片查看器打开。

**编译后运行报 SQLite 错误？**

确保从可执行文件所在目录运行，或双击打开（会自动在 exe 同目录创建 `db/` 文件夹）。

**如何只拉取数据不同步？**

在用户菜单中只选择「拉取最新数据」，不选择「同步到飞书」即可。数据会保存在本地数据库中。
